# Desktop restore Hyprnav runtime orchestration

Restore the Wayland-native Electron bridge for scoped Hyprnav environments, atomic slot synchronization, locking, naming, workspace targets, worktree terminals, favorite editors, shell commands, no-op bindings, and external Corkdiff commands.

Worktree terminals launch or focus per-worktree Ghostty sessions through Hyprnav and are staged into packaged desktop builds. Runtime path resolution supports both electron-builder resources and the Nix app-root layout. Unavailable runtime dependencies return explicit status results instead of rolling back saved configuration.

The active primary-local renderer publishes and locks the current thread environment, retries transient failures without letting superseded threads relock themselves, and retains removed-slot/name cleanup across attempted, acknowledged, and restarted renderer state. A semantic request fingerprint prevents projection object churn from restarting publication. Desktop sync reports the exact scopes it applied, so stale-worktree recovery acknowledges the project fallback without discarding pending worktree or thread cleanup. Persisted publication history contains only slot numbers and name-presence flags, never commands or credentials. It refreshes Corkdiff websocket tickets before expiry through the plugin-redacted `token` query parameter. Hyprnav-launched Corkdiff listens on a stable per-thread Neovim socket so refreshed connections update the running process, while identity-matched stale sockets are recovered before future launches. Browser, remote threads, and desktop platforms without the optional Hyprnav bridge do not start the runtime.
